「不信的時候最準」——這句話是心理學，不是玄學

「不信的時候最準」——這句話是心理學，不是玄學

上週三下午，一位做五金貿易的老闆在我桌子對面搖頭，說他昨晚隨便抽了一張卦，本來是想證明易經是假的，結果那張「蹇卦」把他這半年在馬來西亞設廠遇到的每一道坎都講完了。他說：「Timmy 老師，我本來是要笑它的，結果笑不出來。」

我不覺得這是神蹟。我反而覺得，這才是「不信的時候最準」這句話背後真正有意思的地方。

先把話講清楚。「不信的時候最準」不是說你心裡虔誠就會失靈、抱著懷疑就會靈驗——那是把事情講反了。真正的邏輯是：當你不信的時候，你不會預先設定答案；當你不預設答案，你才看得見卦辭真正在講什麼。

你信的時候會怎樣？我陪過太多老闆抽卦，有些人手還沒放下，腦子裡已經在想「這卦是不是在叫我簽那個合約」。他帶著期待去讀卦辭，卦辭裡任何一句模糊的話，他都會自動往自己想要的方向翻譯。這在心理學有個名字叫「確認偏誤」（confirmation bias）。你信的越深，偏誤越大。

不信的人相反。他帶著「我倒要看你怎麼唬我」的心態去讀，每一句他都在挑戰、在核對、在對照自己的處境。這種讀法，反而讓卦辭的資訊量被完整吸收進去。

這跟企業 AI 導入其實是同一件事

我做 Claude Code 教學這幾年，看過一個很有趣的現象。那些一開始就對 AI 充滿信仰的老闆，導入效果往往最差。為什麼？因為他們不會追問。模型吐什麼，他就信什麼。上個月有一家做烘焙的客戶，Claude 幫他擬的客服回覆稿裡出現一個根本不存在的產品規格，他照貼到官網去，三天後被客人抓到。

反而是那些一開始覺得「AI 沒那麼神吧」的老闆，會逐字檢查輸出、會問「你這個數據從哪來」、會用十年行業經驗去核對模型講的話。結果呢？他們反而把 Claude Code 用得最深，因為他們用的是「協作」的模式，不是「信仰」的模式。

所以「不信的時候最準」，放在易經是準，放在 AI 也是準。都是同一個道理——保持距離才看得清楚。

我自己也有切身經驗。2024 年初我陪一家做車用線束的中小企業主做決策，他要不要跟一個新客戶合作。他抽到「履卦」，卦辭是「履虎尾，不咥人，亨」。如果他當時深信易經，他大概會解讀成「走在老虎尾巴上但不會被咬——所以這個險可以冒」。但那天他狀態很懷疑，他一邊讀一邊說：「履虎尾就是走在危險上，這卦根本在警告我吧？」他最後沒簽。三個月後那個客戶爆雷，欠了供應鏈上下游一大筆。

同一句卦辭，信的人讀成「放心做」，不信的人讀成「小心做」。哪個準？當然是後者——因為後者沒有被期待污染閱讀。

那為什麼還要學易經？

有人會問：既然「不信最準」，那乾脆別學。這就錯了。

不信不等於不懂。你得先懂得夠深，才能在不信的狀態下讀出卦辭的層次。一個從沒讀過蹇卦的人，就算他帶著最大的懷疑抽到蹇卦，他也看不出那張卦在講什麼——因為他連「蹇」是「難行」都不知道。

真正的準，是「懂 + 不執著」。懂讓你有解讀能力，不執著讓你保留判斷空間。這兩個加起來，你才會在那一刻聽見卦辭真正對你講的那句話。

企業 AI 導入的邏輯一模一樣。你要懂模型的能力邊界（不然你會以為它無所不能或一無是處），但你不能迷信它的每一句輸出（不然你會變成照貼的人）。懂 + 保持距離 = 用得深。

說到底，「不信的時候最準」不是在講玄學，是在講認知。它提醒你：你看見的，常常是你想看見的。把這層濾鏡拿掉，事情本來的樣子才會浮出來。卦是這樣，AI 的回覆是這樣，連客戶給你的反饋也是這樣。

這大概是六十四卦藏得最深、也最日常的一課。

常見問題

那我抽卦到底要不要信？我每次抽完都在糾結這個。
信到願意好好讀卦辭，不信到不會照單全收。大概就是這個分寸。如果你抽完立刻想「這卦是不是叫我做 X」，那你已經信過頭了；如果你抽完直接笑一笑關上書，那你又不信到沒吸收任何東西。中間那個「認真讀但保留判斷」的狀態，才是卦真正在用的時候。
用 Claude Code 處理公司事務，也要「不信」嗎？那還用它幹嘛？
不是不信它能幫你，是不信它「不會錯」。我自己用 Claude Code 拆 SOP、寫客服腳本、跑報表，每一份輸出我都會當草稿看，不會當終稿。它幫我省掉七成的起稿時間，但那最後三成的核對才是我真正賺錢的地方。信它會加速你；核對它讓你不出事。兩個都要。
易經跟 AI 放在一起討論，會不會太牽強？
一開始我也擔心這件事。但做了幾年後我反而覺得，這兩個東西背後都在處理同一件事——**在資訊不完整的時候怎麼做決策**。AI 幫你把可以量化的部分算清楚，易經幫你把不能量化的那塊（時機、人心、自己的狀態）照一下鏡子。中小企業主真正卡住的從來不是數據，是那個「該不該動」的當下。這兩套工具剛好一個管左腦、一個管右腦，我覺得不牽強，反而是互補。
