兩套工具，一張桌子

上週一家做塑膠零組件的工廠老闆問我：你到底是顧問還是命理師？

我想了兩秒，說：都不完全是。

命理師告訴你今年財運旺、下半年謹慎。顧問給你一張 ROI 試算表，說導入 AI 三個月可以省多少人事成本。但他現在的狀況——三個兒子對數位轉型的態度各不相同，老員工觀望，上個月的訂單剛回升但還沒穩——這些，試算表算不出來。

那個算不出來的部分，才是我真正在做的東西。

兩套工具，一張桌子

我的工作現場很奇特。筆電開著 Claude Code，幫老闆把客服回覆的 SOP 拆成可以跑的自動化流程；桌角放著一本被翻到封面快脫落的《周易》。不是因為我覺得這兩個東西很配，而是因為這幾年下來，我發現中小企業主真正卡住的地方，不是工具不夠好，是那個「這一步到底該不該踏出去」的糾結。

技術能幫你算成本、算效率、算時程。但告訴不了你：現在動，你的組織承受得住嗎？家族裡的氣，是聚的還是散的？

易經不是算命。我說清楚這點，因為很多人一聽「易經」就想到問卦、燒香、看流年。我用易經的方式更像一個思考框架——它逼你把當下這個局的動態結構看清楚：哪些力量在推進、哪些在阻攔、你現在站的位置是強還是弱、時機是到還是差那麼一口氣。

六十四卦裡，最讓我反覆想起的是「屯卦」。屯，難也，初生之艱。上面是雷，下面是水。很多力量在同時運作，但方向還沒定。我陪中小企業主走 Claude Code 導入，有時候他們的局就是這樣——方向對、資源也有，但組織還沒到可以動的那個點。這個時候你硬推，人走一半，氣就散了。

一家充電線貿易商的故事

去年十月，一家做充電線周邊產品的貿易商找我，說要全面導入 AI 客服自動化。老闆娘很積極，傳給我一份六頁的需求文件，要我幫她評估方案。

我花了四十七分鐘讀完，然後打電話，問了一個問題：你們現在的客服是誰在負責？

她說：一個做了十二年的老員工，剛生完小孩回來上班。

我問：她知道這個計畫嗎？

電話那頭沉默了大概三秒。

這個問題本來不在我的「顧問清單」上。但用易經的思路想這個局：有志有資，但內部陣腳未穩。這個時候不是不能動，是順序要對。

我建議她：先讓那位老員工主導 AI 工具的試跑期，而不是繞過她直接推。三個月後，老闆娘告訴我，那個員工現在是公司裡最積極幫大家學 Claude Code 工具的人。

---

技術那一塊，Claude Code 很實在——客服腳本、訂單彙整、客戶問卷分析，這些重複且可結構化的事，交給它就好。它不累、不需要培訓時間、也不會因為哪天心情不好出錯。

但那個「時機」「人心」「組織當下的能量狀態」——這不是工具的事。這是幾千年人類觀察行為模式累積下來的事。

我只是同時把兩套東西放在桌上用。不宣稱哪個更厲害，也不需要你信卦。你只要願意承認：在你的 ROI 試算表旁邊，有一半的問題是它算不到的。

承認這一半，我們就有話說了。

常見問題

懂易經的顧問跟一般 AI 導入顧問，實際差別在哪？
一般 AI 顧問幫你評估工具、設計導入流程、算投資回報。這些我也做。差別在我會多問一層：你的組織現在有沒有能量承接這個變化？有沒有人會因為這件事覺得自己被邊緣化、或悄悄擔心自己的位置？很多企業 AI 導入的案子最後沒跑起來，不是工具選錯，是人心問題在第三個月才爆出來。易經的訓練讓我對這個「當下動態」比較敏感，通常在啟動前就會想到要問。
企業 AI 導入跟易經，真的能用在同一個決策上嗎？
可以，但要分工清楚。Claude Code 負責你能量化的那一半——分析數據、整理 SOP、產出報表。易經負責你量化不了的那一半——這個時機，你的組織有沒有條件承接？哪個方向阻力最小？用同一個問題問兩套工具，你能看到的面向就不一樣了。
我不信命理，適合找這種結合易經的顧問嗎？
完全適合。我自己也不把易經當命理用，它更接近一套觀察框架——幫你把「當下局勢的動態結構」看清楚。你不用信卦辭，你只需要願意在試算表之外，花半小時把組織裡的力量關係想一想：誰在推、誰在觀望、誰有個說不出口的擔心。這件事，信不信易經都能做，只是我用易經的框架做起來更有效率。
