老闆兼 AI 負責人,不是學不學得會的問題

上週三下午,一位做機械手工具的老闆在我辦公室坐下來第一句話是:「我兒子說我太老了學不來,叫我把 AI 這件事交給他。可是我心裡就是不踏實。」

他六十二歲,公司年營收八千多萬,手下三十七個人。兒子三十一歲,海外留學回來,英文流利。從表面看,讓兒子接手 AI 導入再合理不過。

但我跟他說:你不踏實是對的。

老闆兼 AI 負責人,不是學不學得會的問題

很多人把這件事想成技術門檻。其實不是。Claude Code 也好、自動化流程也好,真正難的從來不是工具本身——我陪過一個快七十歲的老闆,兩週就上手 Claude Code 寫客服回覆模板,比他三十幾歲的特助還快。

難的是「決定要不要做、做到哪個程度、什麼時候停」。

這位機械工具老闆的兒子很聰明,但兒子看的是「這個流程能不能自動化」。老闆看的是「自動化之後,跟了我十五年的廠長會不會覺得自己被取代」。

這兩種視角都對,但只有一個人有權力做最後那個取捨。那個人就是老闆。

我看過交出去之後出事的三種狀況

第一種,交給技術出身的二代或主管。他們很快就把流程跑起來,但選錯了戰場——把自動化用在最能省錢的地方,而不是最該動的地方。一家做食品代工的客戶,二代花了四個月把訂單自動化打通,結果熟客流失了 12%,因為老闆原本每週親自打電話追單的那個動作,在他們眼中是「沒效率的人工流程」,被砍掉了。

第二種,交給外部顧問或 IT 公司。看起來最省事,但顧問離開之後,公司內部沒有人真的懂這套東西在做什麼。出問題的時候,老闆只能再付一次錢。

第三種,交給員工內部小組。動機都對,但因為老闆沒進場,小組的決策只能在「不得罪任何人」的範圍內打轉。AI 該替換掉的低價值工作,沒人敢提。

易經裡有個卦,叫「乾」

乾卦六爻,從潛龍勿用一路到亢龍有悔。中間有一爻叫「九五,飛龍在天」——通常被理解成大展身手的時刻。

但我每次跟老闆讀這一爻,都會多講一句:飛龍在天的前提,是這條龍知道自己在飛什麼方向。

企業 AI 導入這件事,你可以把執行交出去,但「方向感」這個東西,是交不出去的。員工不會比你更懂你的客戶為什麼留下來、廠長為什麼還沒退休、那個總是抱怨的老客戶為什麼其實是公司的命脈。

這些東西,AI 模型再強也學不會。Claude Code 可以幫你寫一千封客服信,但不能告訴你哪一封信不能寄、為什麼不能寄。

那到底要怎麼分工

我的建議很簡單:老闆當 AI 負責人,但不是技術負責人。

意思是,你要懂到能開會、能判斷、能拍板。不需要懂到能自己寫 prompt、跑流程、debug。

這個程度大概要花你每週三到四小時,持續兩個月。比你想像的少,但比你兒子說的「交給我就好」要多得多。

那位六十二歲的老闆後來怎麼安排的？兒子負責執行,老闆每週三早上開一個小時的會,只做兩件事:一,看上週做了什麼、為什麼;二,決定下週動哪裡、為什麼不動別的。

三個月後他跟我說,他現在比以前更清楚公司哪裡在賺錢、哪裡在漏水。這個東西,以前廠長給他看的報表反而看不出來。

中小企業數位轉型最常見的失敗,不是工具選錯,是把方向盤交給不該交的人。

你可以不會開車,但你不能不知道要去哪裡。

常見問題

我已經五十幾歲了,真的還來得及學 Claude Code 這些東西嗎?
來得及。我帶過最年長的客戶六十八歲,做傳統印刷的,現在每天用 Claude 幫他整理客戶報價邏輯。重點不是學語法,是學「怎麼跟 AI 對話拿到你要的答案」——這跟你帶員工的能力其實是同一件事。如果你能教會一個新人,你就能用好 AI。
可是我兒子/女兒比我懂技術很多,讓他們做不是更快嗎?
讓他們做執行端,絕對更快。但「決定要做什麼」這件事,他們的人生閱歷可能還不到位。我看過太多二代用最新技術跑最快的方向,只是那個方向是錯的。老一輩在現場累積的判斷力,沒有捷徑可以替代。
那我要花多少時間在這上面?會不會把本業都耽誤了?
前兩個月每週三到四小時,主要在學「怎麼判斷」而不是「怎麼操作」。兩個月之後,每週一個小時開會就夠了。比你想像的少很多。如果有人跟你說要全職投入,那不是真的在做企業 AI 導入,是在做 IT 部門。
