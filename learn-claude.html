Module 01 · 初級

Claude 是什麼？跟 ChatGPT 差在哪？

先搞清楚你在跟誰說話，再開口

Claude 是 Anthropic 開發的 AI 助理。你可以直接在瀏覽器打開 claude.ai，不用裝任何東西，跟它用中文對話——它的中文表達能力比很多人想像的好很多。

跟 ChatGPT 比起來，Claude 有幾個地方讓我比較常推薦給不是工程師背景的人用：

📄

長文件處理能力強

你可以把一份合約、一篇報告、甚至整份 SOP 貼進去，請它幫你摘要或找問題。ChatGPT 早期這塊比較弱，Claude 從一開始就設計來處理長文本。

🎯

指令執行比較老實

你說「只要回答三點，不要多說」，它通常就真的只說三點。有時候 AI 喜歡自己發揮，Claude 在這塊比較守規矩。

🗣️

對話感比較自然

用中文問它問題，它的回答比較像一個有在思考的人，不太會出現那種「以下是五個要點：」的官腔格式——除非你要求它這樣。

💡

怎麼開始？打開 claude.ai，用 Google 帳號登入，就可以直接免費用了。免費版每天有使用次數上限；如果你每天都在用，Pro 版（每月約 600 元台幣）值得訂閱。

Module 02 · 初級

問對問題的基本功

為什麼你覺得 Claude 沒用，通常是因為問得太模糊

很多人第一次用 Claude 的感受是「還好啊，跟我想的差不多」。問題通常不在 Claude，而在問題本身。

你給的資訊越具體，它給的答案越有用。這個道理跟你交代工作給員工是一樣的——你說「幫我做個簡報」跟「幫我做一個給客戶的三頁簡報，說明我們這次漲價的原因，語氣要誠懇但堅定」，結果完全不一樣。

❌ 太模糊，結果不實用

「幫我寫一封 email」

✅ 具體，拿到可以直接用的結果

「幫我寫一封給廠商的信，說明這批貨有 3% 瑕疵品，我們要求退換，語氣要專業但不強硬，大約 150 字」

❌ 問題太大，答案都是廢話

「幫我分析我的公司要怎麼做數位轉型」

✅ 縮小範圍，答案有行動性

「我是一家 20 人的製造業，目前訂單管理還靠 LINE 和 Excel。幫我列出導入 AI 工具的三個優先步驟，每個步驟說明理由。」

三個讓問題變好的方法

  • 1
    說明你的身份和情境：「我是做 B2B 業務的業務主管」或「我在準備一個給老闆的報告」，Claude 就會用對的角度幫你。
  • 2
    說清楚你要的格式和長度：「用條列式」「寫成一封信」「100 字內」「表格呈現」——這些指令 Claude 都聽得懂。
  • 3
    說清楚給誰看的：「這是要給不懂技術的老闆看的」或「對象是剛入職的新人」，它會自動調整複雜度和用詞。
完整範例 Prompt
我是一家餐廳的老闆，想對老客戶發一則 LINE 訊息， 告知我們調整了部分菜色定價（平均漲 15%）。 請幫我寫一則約 80 字的訊息， 語氣要誠懇、誠實，不要講太多理由， 結尾給個小優惠提示（如：本月生日客人九折）。

Module 03 · 初級

從今天就能用的 6 個場景

不用學新技能，直接套用這些 Prompt 框架

📝

整理會議記錄

把錄音轉文字的逐字稿或筆記貼進去，請它整理成結構清楚的會議摘要。

Prompt 範例 以下是今天開會的逐字記錄，請整理成：決議事項、待辦事項（含負責人）、下次開會要確認的問題。
✉️

寫信 / 回信

告訴它情況、對象、語氣，讓它起草。你再修改細節，比自己從頭寫快三倍。

Prompt 範例 幫我寫一封婉拒廠商報價的信，對方是長期合作夥伴，原因是這次預算縮減，語氣要有禮貌且留有餘地。
📊

資料摘要與整理

把一篇長報告、一份 PDF 文字、或一大段說明貼進去，請它抓重點。

Prompt 範例 以下是一份產業報告（xxx 字），請用五個條列點說明對我們這種中小型製造業最相關的資訊。
🌐

翻譯與潤稿

不只翻譯，還可以要求它「翻成正式商業用語」或「把這段英文整理成可以貼到 LinkedIn 的格式」。

Prompt 範例 把以下英文合約條款翻成繁體中文，遇到法律術語請保留英文並在括弧內加上中文說明。
📋

草擬計畫或 SOP

說明你要解決的問題和限制條件，請它幫你起草第一版。你再補細節，比空白頁開始快多了。

Prompt 範例 我要做一個新員工前 30 天的到職 SOP，公司是 15 人的貿易公司。請列出七個階段，每個階段說明目標和主要任務。
🔍

解釋你不懂的東西

看到不懂的術語、條款、或新聞，直接貼給 Claude，請它用你聽得懂的語言解釋。

Prompt 範例 幫我解釋「責任保險的除外條款」是什麼意思，我是第一次看保險合約的老闆，請用日常語言說明。

Module 04 · 初級

用之前要知道的三件事

避開常見的錯誤認知

⚠️

不要輸入公司機密。你跟 Claude 說的話，預設情況下 Anthropic 可能用來改善模型。合約內容、客戶名單、財務數字、員工個資——這些不要貼進去。如果公司有嚴格的資安政策，先確認再用。

⚠️

Claude 會說錯話，你要驗證。它有時候會生成看起來很合理、但根本不存在的數據或法規。數字、日期、法律條文——一定要自己確認。把它當「有水準的草稿機」，不是「百科全書」。

✅

不滿意就直接說。Claude 不會生氣。「這個寫得太官腔，重寫一版口語一點的」「前兩點好，第三點換成講風險」——直接說它就會改。不用客氣，也不需要重新問一次。

💡

用完就關，不會記住你。每次開新對話，Claude 不記得上次你說過什麼。如果你有特定的背景資訊想讓它每次都知道（例如：公司規模、產業、你的職位），可以存一段說明，每次開頭貼進去。

Module 01 · 中級

給 Claude 一個角色（System Prompt）

讓它每次開口就知道自己在扮演什麼

初級的問法是「每次臨時說」。中級是「一開始就設定好角色」，之後每次問它，它都帶著那個角色的知識和語氣回答你。

最簡單的做法：在對話的開頭，先說一段「角色設定」。

角色設定範例：業務顧問
你是一個在製造業服務超過 15 年的資深業務顧問， 熟悉 B2B 大客戶談判和長期客戶關係維護。 我接下來問你的問題，請用實際業務場景的角度回答， 不要給理論性的建議，要給可以馬上試的具體做法。
角色設定範例：人資主管
你是一個有 10 年台灣中小企業人資管理經驗的 HR 主管， 熟悉勞基法和實務操作之間的落差。 我接下來問你的問題，請用台灣中小企業實際情況回答， 法律條文要說清楚，但也告訴我實務上通常怎麼處理。

角色設定的三個要素

  • 1
    身份：這個角色是誰？做什麼的？幾年經驗？讓 Claude 知道從什麼視角說話。
  • 2
    知識背景：它對什麼特別熟悉？這樣它的回答會更精準，不會給通用的廢話。
  • 3
    回答風格：具體還是概念性？直接還是帶分析？幾個要點？告訴它你要的格式。
💡

把你常用的角色設定存在手機備忘錄或電腦記事本裡。下次開新對話，直接貼進去，不用重寫。

Module 02 · 中級

要求結構化輸出

讓 Claude 給你可以直接用的東西，而不是再改一遍

Claude 很容易給你一大段文字，但你真正需要的可能是一份表格、一個清單、或者一個可以貼進 PPT 的結構。這件事你要主動要求。

❌ 輸出是一大段，還要自己拆

「分析一下這三家廠商的優缺點」

✅ 直接出表格，可以貼進簡報

「用表格比較這三家廠商，欄位包含：價格、交期、品質穩定度、售後服務，每格用一到兩句評語。」

常用的輸出格式指令

📋

清單格式

Prompt 指令 請用條列式列出，每點不超過兩句話，不要加說明段落。
📊

表格格式

Prompt 指令 請用 Markdown 表格呈現，欄位是：項目、優點、缺點、適合情境。
📄

報告格式

Prompt 指令 用以下結構輸出：【摘要】一段、【主要發現】三點、【建議行動】三點，每個建議附理由。
💬

對話 / 腳本格式

Prompt 指令 請以「業務：」和「客戶：」交替的對話格式呈現，模擬客戶提出價格異議的情境。

Module 03 · 中級

多輪對話技巧

不要每次都重新問，學會在同一個對話裡繼續深挖

很多人用 Claude 的方式是：問一個問題，拿到答案，關掉。下次有問題再開新的。這樣很浪費。

Claude 在同一個對話視窗裡，會記得你之前說的所有東西。你可以在它的回答上直接繼續挖：

  • 1
    局部修改：「第二點寫得太官方，改成口語一點，但意思不要變。」
  • 2
    追問：「剛才說的第三個建議，可以舉一個台灣中小企業的真實例子嗎？」
  • 3
    加入新條件：「如果我的預算只有 10 萬，剛才那個計畫要怎麼調整？」
  • 4
    要求反向思考：「你剛才給了支持這個決策的理由，現在給我三個反對的理由。」
多輪對話範例（節錄）
你：幫我分析這份客戶提案，找出三個潛在風險。 Claude：[給出三個風險分析] 你：第二個風險最讓我擔心。這個問題在合約上 要怎麼保護自己？請給我具體的合約條款方向。 Claude：[針對第二個風險的合約保護建議] 你：把這些合約建議整理成一封給律師的說明信， 讓律師看得懂我想保護什麼。
✅

一個好的對話可以推進很深。我有時候一個對話開著處理同一個問題超過一小時，從分析到起草文件到模擬反駁，全在同一個視窗裡完成。

Module 04 · 中級

老闆最常用的商業應用場景

這些不是範本，是實際測試過有效的 Prompt 結構

⚔️

競爭對手分析

把你知道的競爭對手資訊整理好貼進去，請它幫你找差異化機會。

Prompt 框架 我是做 [產業] 的，主要競爭對手是 A、B、C。以下是我知道的他們的狀況：[資料]。請分析我們的差異化機會，不要給通用建議，要基於我提供的資料。
📈

SWOT 分析

說明你的公司現況，讓它幫你做 SWOT，再追問哪個 SO 策略最可行。

Prompt 框架 我的公司：[基本狀況]。市場環境：[目前的挑戰和機會]。請做一個 SWOT 分析，每項不超過三點，最後給一個最值得投入的 SO 策略建議。
📝

合約摘要與風險標注

把合約文字貼進去，請它找出重要條款和潛在風險，再決定哪些要問律師。

Prompt 框架 以下是一份採購合約，請：1) 摘要主要條款 2) 標出對我方（買方）不利的條款 3) 列出我應該跟律師確認的問題。
🎤

簡報大綱 / 演講稿

告訴它對象、時間、你想傳達的核心訊息，讓它起草結構，你再補細節。

Prompt 框架 我要對 [對象] 做一個 [時間] 分鐘的簡報，主題是 [主題]，核心訊息是 [訊息]。請給我一個三段式的簡報大綱，每段說明應該講什麼、為什麼要在這個位置講。
🤝

談判情境模擬

請 Claude 扮演你的客戶或廠商，練習你的談判說法，然後請它給你反饋。

Prompt 框架 請扮演一個對我們報價有壓力的採購主管。我會提出我的方案，請你從採購角度提出反對和異議，讓我練習回應。我說「結束」才停。
🔧

問題診斷 / 決策輔助

描述你卡住的問題，請它幫你拆解為什麼卡、有哪些選項、每個選項的代價。

Prompt 框架 我在 [問題描述] 上卡住了。背景：[情境]。請幫我：1) 分析這個問題的根本原因 2) 列出三個可能的解決方向 3) 每個方向的主要風險。

Module 05 · 中級

先定框架，再讓 AI 加速

昆廷老師在顧問工作裡用的方法

我常說：AI 不是在替你做決定，是在你想清楚框架之後，把分析速度拉快十倍。

最常見的錯誤用法是：問題還沒搞清楚，就把 Claude 當答案機在用。結果它給你一堆分析，但全部都在解答一個錯的問題。

我在陪客戶做 AI 導入的時候，常常先做一個問題定框架的步驟，才讓 Claude 進場。這個框架來自易經的問題意識——不是要算命，而是強迫自己把「我在哪個局、我真正要解決的是什麼」先說清楚。

決策輔助 Prompt 框架（昆廷老師版）
背景：[我目前的處境，盡量具體，包含數字] 真正的問題：[我認為核心問題是什麼] 我考慮的方向：[已經想到的選項，即使不完整也寫出來] 我的限制：[時間、預算、人力、不能動的因素] 請幫我： 1. 確認我理解的「真正問題」是否準確，或者有沒有我忽略的假設 2. 針對我的選項，分析各自的主要風險 3. 如果要做決定，你建議我優先釐清哪一個問題
💡

這個框架最有用的地方是第一步：確認問題定義。很多時候 Claude 會回你「你說的問題可能不是根本原因，真正的問題可能是...」——這個反饋本身就值回票價了。

✅

如果你想把這個方法應用在更複雜的組織決策情境，或者想進一步了解怎麼把易經問題框架和 AI 分析結合在一起，可以直接聯絡我。這是我在顧問工作裡最常帶企業主練的一件事。
